# ADR-010: Adopt an Abstracted Data & File Storage Strategy

**Status:** Accepted

**Date:** 2026-07-02

---

# Context

WBOS stores two categories of information:

**Structured Business Data**

* Products
* Customers
* Invoices
* Payments
* Inventory

and

**Binary Assets**

* Product Images
* Supplier Documents
* Purchase Invoices
* Shipping Documents
* PDF Exports
* Future Barcode Images

These have very different storage requirements.

---

# Decision

Business data will always be stored in PostgreSQL.

Binary assets will be stored through an abstract Storage Provider.

The application must never depend directly on the underlying storage implementation.

---

# Storage Philosophy

Business data belongs in the database.

Files belong in object storage.

A database is not a file system.

A file system is not a database.

Each technology should do what it does best.

---

# Storage Abstraction

Every file operation should pass through a Storage Service.

Example:

```text
Application

↓

Storage Service

↓

Storage Provider

↓

Local Disk / S3 / MinIO
```

Business logic should never know where files are physically stored.

---

# Phase 1

Initial storage provider:

* Local Docker volume

Suitable for:

* Homelab deployment
* Development
* Small business operations

---

# Future Providers

The architecture should support:

* MinIO
* Amazon S3
* Cloudflare R2
* Azure Blob Storage
* Google Cloud Storage

Switching providers should require configuration rather than application redesign.

---

# Metadata

The database stores metadata, not file contents.

Examples:

* File Name
* MIME Type
* Size
* Owner
* Storage Key
* Uploaded By
* Upload Date

The binary data remains outside PostgreSQL.

---

# Why Not Store Files in PostgreSQL?

Rejected.

Advantages:

* Simpler backups.

Disadvantages:

* Larger database size.
* Slower backups.
* Poor scalability.
* Reduced flexibility.

PostgreSQL should remain focused on relational data.

---

# Architectural Rules

* Business data belongs in PostgreSQL.
* Binary files belong in Storage Providers.
* Storage is abstracted behind a service.
* Business logic never depends on storage implementation.
* File metadata belongs in the database.

---

# Final Decision

WBOS separates structured data from binary assets through an abstract storage layer.

This keeps the application portable, scalable, and infrastructure-independent.
